** 3-tier deployment **

Title: Deploying a Three-Tier Application with React.js, Node.js, MongoDB using GitHub, Docker, Jenkins CI/CD, ECR, EKS

Description: This project demonstrates the deployment process of a three-tier application comprising React.js for the front-end, Node.js for the back-end, and MongoDB for the database layer. Utilizing a combination of GitHub, Docker, Jenkins CI/CD, Amazon ECR, and Amazon EKS, this deployment pipeline ensures scalability, reliability, and automation throughout the software development lifecycle.

1. GitHub: The project leverages GitHub for version control, enabling collaborative development and version tracking among the development team. Developers can work concurrently on different tiers of the application, managing changes through branches and pull requests.

2. Docker and Amazon ECR: Docker is utilized for containerization, allowing each tier of the application to be encapsulated into separate containers. Amazon Elastic Container Registry (ECR) is employed as a fully managed Docker container registry within the AWS ecosystem, ensuring secure storage and retrieval of Docker images for deployment.

3. Docker and Amazon ECR: Docker is utilized for containerization, allowing each tier of the application to be encapsulated into separate containers. Amazon Elastic Container Registry (ECR) is employed as a fully managed Docker container registry within the AWS ecosystem, ensuring secure storage and retrieval of Docker images for deployment.

4. Jenkins CI/CD: Jenkins serves as the core for continuous integration and continuous deployment pipelines. Upon changes being pushed to the GitHub repository, Jenkins automatically triggers the CI/CD pipeline. The pipeline includes stages for building Docker images for the React.js front-end, Node.js back-end, and MongoDB, running tests, pushing images to Amazon ECR, and deploying containers to Amazon EKS clusters.
